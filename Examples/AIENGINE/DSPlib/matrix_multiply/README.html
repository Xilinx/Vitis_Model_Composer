Importing AI Engine matrix multiply from DSPLib as a block into Model Composer

This example demonstrates importing a matrix multiply function as a block into Model Composer and comparing the results to Simulink matrix multiply. In this example, we use the import graph block to import the matrix multiply function.

Knowledeg nuggets

Note the usage of the environment variable in MATLAB to more easily set the paths in the AIE Graph block mask.

Note this example is using the AI Engine DSP library that is shipping with Vitis Model Composer. We are using the command setenv("DSPLIBROOT",xmcLibraryPath('get','dsplib')) to set the DSPLIBROOT environmental variable.

Note we are importing matrix multiply as a graph.
